An OOP principle that states that there should be one interface and different methods that this interface calls depending on what you pass to this interface. There are two types
### Compile-time
The decision about which function to use is made by the compiler and is hence faster. Typically achieved by function overloading or generics / templates. Also called static polymorphism

### Runtime
The decision about which function to run is decided at runtime and is therefore slower. Achieved by inheritance and method overriding. Also called dynamic polymorphism
