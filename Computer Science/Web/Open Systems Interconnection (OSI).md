### Application layer
The pinnacle of the model. Where all the bottom layers are finally put to use to create, well, anything. But usually it's a protocol like VoIP, HTTP, SSH, FTP and DNS among many others

### Presentation layer
Responsible for getting information ready for the Application or the Session layers, this layer focuses on de-encryption, looking for errors and other stuff

### Session layer
Exactly what it sounds like — manages sessions between computers

### Transport layer
This layer provides the means by which data can be transferred. Here live, for example, [[IP]] and [[UDP]]

### Network layer
Kinda like the Transport layer but for smaller and atomic parts

### Data Link layer
This layer provides connections from one node to another. This is where [[MAC Address]] lives

### Physical layer
The hardware
